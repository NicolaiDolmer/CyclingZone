1. **Største close-batch (15) leveret med høj konfidens:** Aggressive default + label-cleanup pattern + NOW.md pre-flagging gjorde det muligt at lukke 15 issues uden tilbageblik.
2. **Codex' weekly routine pre-screened:** Routine producerede #660 + recommended close på #578 → manuel skill kunne starte med shortlist i stedet for at lave hele scan-fra-bunden.
3. **Persistent scoring-scripts virker:** `score_done.py`/`crossref.py`/`labelcheck.py`/`staleblocked.py` kørte direkte uden inline-rewrite (Python TEMP-path løst).
4. **Label-cleanup-på-close pattern:** 4 todo+done konflikter samt #348 manual:user fjernet + lukket samme handling. Forhindrede gentag af "skipped pga. label"-tracking fra 2026-05-25.

### Hvad var tvivl
1. **#327 Phase 5 PR merged men issue carry-forward:** Multi-phase issues hvor PR-merge ≠ issue-done er edge case. Skill håndterede korrekt via "Pending bruger-action" text-match, men ingen automated rule.
2. **#385 AC6 spun off:** Issue lukket selvom AC6 ikke leveret — accept fordi spin-off til #684 er kanonisk. Men hvis skill kun parser "alle AC met" kunne det misse spin-offs.
3. **#449 scope-flow:** Issue startede som Discord-DM bug, fik B11-merge fra workflow-analyse. Skill holdt issue åben pga. user-verify pending, men scope-vandring er svær at audit.

### Hvad missede vi
- **Ingen new findings i Trin 5 (Epic-rollup):** Som vanligt, info-only kun. Bekræfter lektion 2026-05-23 om at de-prioriterer Trin 5.
- **Brugerverifikation "section no boxes" stiger (+4):** Værd at undersøge om PR-template auto-tilføjer header. Forslag som follow-up issue, ikke skill-edit.

### Hvad var for meget
- **Ingen redundant work denne kørsel:** Persistent scripts + NOW.md pre-flag eliminerer 80% af inline-Python-rewrite.

### Hvad bed mig
- **Script-rename-friktion:** `audit-open.json` vs `audit-open-all.json` + `audit-prs.json` vs `audit-pr-merged.json` + `audit-blocked.json` vs `audit-blocked-issues.json`. Scripts forventer andre navne end mine standard-fetch outputs. **Foreslå edit: standardisér filnavne** mellem Trin 1 og scripts.

### Foreslåede skill-edits

1. **Trin 1 — standardisér filnavne:** Match script-forventninger direkte. Brug `audit-open-all.json` / `audit-pr-merged.json` / `audit-blocked-issues.json` fra første fetch. Sparer 3 `cp`-kald per audit.

2. **Trin 4 — Multi-phase issue detection:** Hvis issue body indeholder "Phase X" + checkbox-status med både ✅ og ⚠️/❌, og latest comment har "spun off til #N" eller "tracked separat" → markér som `MULTI_PHASE_LEGIT_CARRY_FORWARD` i stedet for at score som STRONG/WEAK. Forhindrer false-close-suggest på #327-lignende issues hvor delivery er per-phase.

3. **Trin 3 — Scope-flow detection (info-only):** Hvis issue har modtaget "B11-merge fra #555" eller "merged into denne scope" patterns i comments, flag at scope kan have udvidet. Skill udfører ingen action, men bruger ser scope-vandring eksplicit i artifact. Eksempel #449.
