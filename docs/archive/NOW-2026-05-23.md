## Session 2026-05-23-G — #577 Idempotency-fix for negative-interest debit KOMPLET

`processTeamSeasonPayroll()` debiterede negativ-balance-rente uden idempotency-guard — ved cron-retry ville renten blive trukket flere gange (og hvert gentagende beløb ville være større fordi balancen er mere negativ). Fix: tilføjet `idempotent: true` + `idempotencyKey: negative_interest:${team.id}:${seasonId}` til interest-debit, konsistent med salary-debits eksisterende pattern. `uniq_finance_idempotency_key` DB-index (2026-05-09) dækker dette uden ny migration. Ny test i `economyInvariants.test.js` (test 10) verificerer at gentaget kørsel ikke dobbelt-debiterer. Refs [#577](https://github.com/NicolaiDolmer/CyclingZone/issues/577).
