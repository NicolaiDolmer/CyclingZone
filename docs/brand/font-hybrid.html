#481 · Hybrid font roles · Where Inter Tight earns its place

DM Sans for reading, Inter Tight for data

Your instinct: keep DM Sans (warm, already live) for words you read, and bring in Inter Tight where it is genuinely better. It is better at one thing above all: data you scan. Below, each data component is shown DM Sans (left) vs Inter Tight (right). Numbers use tabular figures in both, so the difference you see is the letterforms, not the spacing.

Rule of thumb: DM Sans = prose & reading. Inter Tight = numbers, tables, dense functional chrome. Bebas stays the display/wordmark font in all cases.

5 · Prose stays DM Sans
Where you actually read, DM Sans wins on warmth. Founder copy, help, onboarding, modals, patch notes.
A fair premium conversation

Cycling Zone has been free since the start, and the competitive game will stay free. I am exploring premium, but the rule is simple: you cannot pay for better riders, faster training, or better results. If premium happens, it is for identity, analytics, and backing the project, never for advantages. I would rather learn the hard truth now than build the wrong thing.

My recommendation

Yes, do the hybrid, and scope it tightly. Use Inter Tight for: stat tables, rankings/leaderboards, big-number KPIs, prices/watts/FTP, dates & timers, and dense chrome (badges, table headers, compact nav). Keep DM Sans for: everything you read in sentences (body copy, help, onboarding, modals, patch notes, worded buttons).

The honest trade-off: this makes a 3-font system (Bebas display + DM Sans reading + Inter Tight data). That is more to manage than one font, and the DM Sans vs Inter Tight difference is subtle in prose, so the split must follow a clear rule (numbers/data vs words) or it looks accidental. Done with that rule, it is a real upgrade: warmth where you read, precision where you scan. It also keeps the brand-sibling logic intact, because Inter Tight is the same font as the CZ favicon.

Cleanest implementation: one CSS class (e.g. .font-data / a Tailwind font-data token) applied to numeric/data components, DM Sans as the default body. That makes the rule enforceable, not vibes.
